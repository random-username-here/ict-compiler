# Some hungarian notation conventions

Well, original form to note type (`i` for ints, etc) is not
used here. Here I use these prefixes for semantic info:

 - `l_` for static functions
 - `m_` for private elements
 - `p_` for protected elements
 - `g_` for global variables
 - `i_` for implementation parts in header files.

I may sometimes use them for typing, when I have 
2 variables with same thing, just in different types.
For example `v_` for SIMD variable. But this is rare.
